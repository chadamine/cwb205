Chapter 2 - Lexical Structure

2.1 - Character Set

JS uses Unicode char set as it can represent virtually every written language in common use.
Each character is represented as 2 bytes as opposed to 8 bits in other languages.

2.2 - Case Sensitivity

JS is a case-sensitive language.

2.3 - Whitespace and Line Breaks

JS ignores spaces, tabs, and newlines between tokens (except those in strings/regular expression literals.

2.4 - Optional Semicolons

Semicolons may be omitted if each statement is placed on a seperate line:

a = 3; a = 4;

is equivalent to:

a = 3 a = 4

However, for the following statement:

a = 3; b = 4;

the first semicolon is required. It is a good programming practice to use semicolons liberally
JS will insert semicolons when it thinks one was omitted if lines of code are inappropriately broken:

return true;

will be interpreted as:

return; true;

2.5 - Comments

C/C++ style comments (// and /* */) are supported in JS.

2.6 - Literals

Literals are data values that appear directly in the program, and include the following:
Integrals, floats, "strings" or 'strings', bool, regular expressions (/javascript/gi), and null.
Expressions that serve as array and object literals are supported in ECMAScript v3:

{ x:1, y:2 }
[1,2,3,4,5]

2.7 - Identifiers

Follows same rules as other languages:
Must start with letter, underscore, or dollar (but avoid these)
ECMAScript v3 can contain letters and digits from complete Unicode char set.
It also allows escape characters (\u03c0)

2.8 - Reserved Keywords

Reserved JavaScript keywords:
break, case, catch, continue, default, delete, do, else, false, finally, for, function, if, in, instanceof, new, null, return, switch, this, throw, true, try, typeof, var, void, while, with


Words reserved for ECMA extensions:
abstract, boolean, bye, char, class, const, debugger, double, enum, export, extends, final, float, goto, implements, import, int, interface, long, native, package, private, protected, public, short, static, super, synchronized, throws, transient, volatile

ECMAScript v4 potential reserved words:
as, is, namespace, use

Other identifiers to avoid (global variables and predefined functions) in ECMAScript v3:
arguments, Array, Boolean, Date, decodeURI, decodeURIComponent, encodeURI, Error, escape, eval, EvalError, Function, Infinity, isFinite, isNaN, Math, NaN, Number, Object, parseFloat, parseInt, RangeError, ReferenceError, RegExp, String, SyntaxError, TypeError, undefined, unescape, URIError

3.1 - Numbers

No distinction made between integer and float values, all numbers represented as float.
Numbers represented using 64-bit floating-point format defined by IEEE 754 standard.
Range: +/-5x10^-324 to +/-1.7976931348623157x10^308 (double type range).

3.1.1 - Integer Literals

Values between 9007199254740992 (-2^53) and 9007199254740992 (2^53) inclusive.

Certain integer operations (i.e. bitwise operators) performed on 32-bit ints, -2147483648 (-2^31) to 2147483647 (2^31-1

3.1.2 - Hexadecimal and Octal Literals

Hexadecimal (base-16) values recognized.

0xff // 15 * 16 + 15 = 255 (base 10)
0xCAFE911

ECMAScript standard does not support octal (base-8) format, but some implementations of JS do.

0377 // 3 * 64 + 7 * 8 + 7 = 255 (base 10)

3.1.3 - Floating-Point Literals

floats are real numbers/values: integral.fractional
floats can be represented using exponential notation: [integral][.fractional](E|e)[(+|-)]integerExponent]

3.1.4 - Working with Numbers

Basic arithemetic operators: + - * /

Mathematical functions provided by Math object:

sine_of_x = Math.sin(x); hypot = Math.sqrt(x * x + y * y);

3.1.5 - Special Numeric Values

Infinity is a special value representing floating-point values larger than the largest representable finite number
-Infinity (negative infinity) represents floats smaller than the smallest possible finite number

NaN (not-a-number) is an undefined result or error (such as 0 / 0)
NaN does not compare equal to any number, including itself

3.2 - Strings

JS does not use char, use a string instead

3.2.1 - String Literals

Double-quote characters can be contained within single-quote delimited strings
Single-quote characters can be contained within double quote delimited strings
String literals must be written on a single line
Choosing a standard of single or double quotes for JavaScript or HTML is helpful:

<a href="" onclick="alert('Thank you')">Click Me</a>

3.2.2 - Escape Sequences in String Literals

Escape characters are used in JS as they are in other languages
Latin-1 characters can be specified with \xXX where XX are hex digits or \XXX where XXX are octal digits between 1 and 377 (not supported by ECMAScript v3)
Unicode characters can be specified by \uXXXX where XXXX are hex chars
Other escape sequences:
\0 (\u0000)
\b (\u0008)
\t (\u0009)
\n (\u000A)
\v (\u000B)
\f (\u000C)
\r (\u000D)
\" (\u0022)
\' (\u0027)
\\ (\u005C)
A backslash not followed by one of the above characters is ignored.

3.2.3 - Working with Strings

String concatenation supported
JS strings and arrays 0-indexed
Strings use object syntax for accessing properties and methods but are not objects.
String length can be determined with length property, accessed as: s.length (s is a string id)
get last character of string s: last_char = s.charAt (s.length - 1)
extract characters from string s: sub = s.substring(1,4);
find position of first letter a in string s: i = s.indexOf('a');
Some implementations support reading individual characters from strings using array notation: last_char = s[s.length - 1]; however this is not supported by ECMAScript v3 and should be avoided

3.3 - Boolean Values

true or false, test: a == 4
Although bool is distinct type instead of simulated as in C, it is easily converted to other data types and is not as pure as Java boolean data type.

3.4 - Functions

Function: piece of executable code, user defined or predefined 

	function square(x) // x is parameter (argument)
	{
		return x*x;
	}

Functions are values that can be manipulated by JS code: they can be stored in variables, arrays, and objects and they can be passed as arguments to other functions.
When a function is assigned to a property of an object it is referred to as a method of that object.

3.4.1 - Function Literals

Function literals are defined in a way similar to function definitions except they need not have names:
function square(x) { return x*x; }
can be defined as
var square = function(x) { return x*x }
without a name.
Functions defined nameless as above are sometimes called lambda functions in homage to LISP programming language which first allowed unnamed functions to be embedded as literal data values.
The argument list and function body can be passed as strings to the Function() constructor:
var square = new Function("x", "return x*x;");
The above technique is often awkward and less efficient than the other ways to define a function.

3.5 - Objects

An object is a collection of named values (properties or sometimes fields)
Properties of objects are referred to as: [object].[propertyName]
Examples: image.width, image.height
Object properties, like JS variables, can contain any type of data including arrays, functions, and other objects.
e.g. document.myform.button in which the button property of an object is stored in the myform property of an object named document.
When a function value is stored in a property of an object it is called a method, the property name becoming the method name.
The method of an object is invoked using the member access operator (.) to extract function value from object
The function is evoked using the () syntax. Invoking the write() method of the Document object:
document.write("this is a test");
Objects can serve as associative arrays (associate arbitrary data values with arbitrary strings.
A string containing the object property name is enclosed in brackets to access the object properties:
image["width"] image["height"]

3.5.1 - Creating Objects

Constructor functions are used to create objects 

	var o = new Object();
	var now = new Date();
	var pattern = new RegExp("\\sjava\\s", "i");

Once a custom object has been created the properties can be used and set: 
	var point = new Object();
	point.x=2.3;
	point.y = -1.2;

3.5.2 - Object Literals

Also called object initializers, object literals contain a comma-separated list of colon-separated property/value pairs all enclosed in curly braces: 

	var point = { x:2.3, y:-1.2 };

Object literals can also be nested: 
	var rectangle = { upperLeft: {x: 2, y: 2 },
										lowerRight: { x: 4, y: 4 }
									};

The property values used in object literals need not be constant (can be arbitrary expressions): 
	var square = { upperLeft: { x:point.x, y:point.y },
									lowerRight: { x:(point.x + side), y:(point.y + side) }};

3.6 - Arrays

An indexed collection of data values.
0 indexed.
May contain any type of JS data: references to other arrays, objects, functions.
Referenced by non-negative integers as opposed to associative arrays mentioned earlier.
Multidimensional arrays not supported except as arrays of arrays.
Elements of arrays in JS need not be of the same type as JS is an untyped language. 

	a[2]	// third element of array

	document.images[1].width	// width property of an object stored in second element of array stored in images property of document object

3.6.1 - Creating Arrays

Arrays can be created with the Array() cunstructor function.

	a[1] = "JavaScript";

Arrays can be initialized by passing array elements to the Array() constructor:

	var a = new Array(1.2, "JavaScript", true, { x:1, y:3 });

Passing a single number to Array() constructor specifies length of array. An array of 10 elements: 

	var a - new Array(10);

3.6.2 - Array Literals

A comma-separated list of values contained in square brackets:

	var a = [1.2, "JavaScript", true, { x:1, y:3 }];

Array literals can be nested:

	var matrix = [[1,2,2], [4,5,6], [7,8,9]];

The elements in array literlas (as with object literals) can be arbitrary expressions:

	var base = 1024;	// Constant

Undefined elements can be included in an array literal. An array of five elements:

	var sparseArray = [1,,,,,,5];

3.7 - null

Special keyword value indicating no value.
Considered special value of object type: a value that represents no object.
null in JS not the same as 0 as in C languages, although sometimes it may be converted as such.

3.8 - Undefined

A variable that has been declared but never assigned a value or nonexistant object property.
Not the same as null, however...

Number, String, and Boolean classes supported along with number, string, boolean data types.

Chapter 4 - Variables

4.1 - Variable Typing

JS is untyped: variable can hold a value of any data type
Values are conveniently and automatically converted from one type to another
Appending a number to a string will convert the number to a string to be appended

4.2 - Variable Declaration

Variables must be declared with the var keyword before being used: var i; var sum;
Multiple variables can be declared with same var keyword: var i, sum;
Variable delcartion can be combined with variable initialization: var message = "hello";
var i = 0, j = 0, k = 0;
Variables that are declared but not initialized contain undefined value
Variables declared with var are permanent and will produce an error if delete used

4.2.1 - Repeated and Omitted Declarations

An error will result from attempting to read the value of an undeclared variable.
A variable can be declared more than once with the var statement.
Variables will be implicitly declared if a value is assigned to an undeclared variable.
Implicitly declared variables are created as global variables even within the body of a function.
The var statement must be used to create a local variable.

4.3 - Variable Scope

Scope of a variable is the region of program in which it is defined.
Global variables have global scope.
variables declared within a function are defined only within the function body, and are local variables with local scope.
Function parameters count as local variables and are defined only within the body of the function

A local variable takes precedence over a global variable within the body of a function.
Declaring a local variable with same name as a global variable will hide the global variable.
Function defintions can be nested each with their own scope, resulting in nested layers of local scope: 

	var scope = "global scope";	// A global variable
	function checkscope() 
	{
		var scope = "local scope";	// A local variable
		function nested() 
		{
			var scope = "nested scope";	// A nested scope of local
			document.write(scope);	// Prints "nested scope"
		}
		nested();
	}
	checkscope();

4.3.1 - No Block Scope

All variables declared in a function are defined throughout the function as there is no block-level scope.
Take caution when used to using block-scope variables such as in for loops, etc. 

	var scope = "global";
	function f() 
	{
		alert(scope);		// Displays "undefined", not "global"
		var scope = "local";	// Variable initialized here but defined everywhere
		alert(scope);		// Displays "local"
	}
	f();

The first alert displays "undefined" because in the var scope is defined throughout function but has not yet been initialized

4.3.2 - Undefined Versus Unassigned

There are two types of "undefined" variables: those never declared, and those declared with no value assignment.
Those with no value assignment should be though of as "unassigned", though they have a undefined value
Those never declared will produce a runtime error if an attempt is made to read their value

4.4 - Primitive Types and Reference Types

Primitive types: numbers, boolean values, null, undefined types
Reference types: objects, arrays, functions
Primitive types havea fixed size in memory: number = 8b, boolean = 1B
Reference types do not havea fixed size, therefore values cannot be stored directly in 8 bytes of memory
Variable stores a reference to the value, usually some form of pointer or memory address.
It is not the data value itself, but tells the variable where to look to find the value. 

	var a = [1,2,3];	// Initialize variable to refer to array (reference to array assigned, not array itself)
	var b = a;		// Copy variable's value to new variable (create new reference)
	a[0] = 99;		// Modify array using original reference
	alert(b);		// Display changed array [99,2,3] using new reference

Variables hold the actual value of primitive types but hold references to values of reference type.
Strings are an unusual case: they are immutable as there is no way to change their contents.

4.5 - Garbage Collection

Storage for strings, objects, and arrays allocated dynamically when size known.
Memory must eventually freed after dynamic allocation, JS relies on garbage collection to do this
This is unlike C/C++ in which memory must be manually freed (onerous and often source of bugs)
JS interpreter detects when objects will not longer be used by the program (object unreachable).

4.6 - Variables as Properties

Variables in JS are fundamentally the same as object properties

4.6.1 - The Global Object

The JS interpreter creates a global object before executing any code, the properties of which are the global variables of JS programs.
Declaration of global JS variables is defining the properties of the global object

Global object initialized by JS interpreter with properties referring to predefined values and functions.
These properties include Infinity, parseInt(), and Math.
In top-level code (outside of a function), the keyword this refers to the global object.
this has a different meaning within functions

In client-side JS the Window object serves as global object for all JS code within browser window it represents
Global Window object has self-referential window property that can be used instead of this
Window object defines core global properties: parseInt() and Math and global client-side properties: navigator and screen

4.6.2 - Local variables: The Call Object

Local variables are properties of an object known as the call object.
The call object has a shorter life than global object but serves the same purpose.
Funciton arguments and local variables stores as properties of call object while funciton body executes
Use of a separate object for local variables allows JS to keep global variables from being overwritten by local variables with same name

4.6.3 - JavaScript Execution Contexts

An execution context is created when the JS interpreter begins to execute a function.
The object in which variables are defined is an important part of this context.
JS code not in a function runs in an execution context that uses the global object for variable definitions.
Every JS function runs in its own unique execution context with its own call object in which local variables are defined.

JS implementations may allow multiple global execution contexts with a different global context.
Each browser window or each frame defines a separate global execution context, each with its own global object.
These client-side global objects have properties linking one another.
This can be demonstrated with a reference to another frame using parent.frames[1]
The frame described above can have a global variable x referenced by the expression parent.frames[0].x in the second frame.
Scripts can run in different global execution contexts and they can refer to each other.
A security consideration with personal data and foreign scripts in seperate frames will be discussed later.

4.7 - Variable Scope Revisited

Every JS execution context has a scope chain associated with it: a list or chain of objects.
The first object in a chain is analyzed first when JS code looks up the value of a variable x(variable name resolution
If the first object has a property named x the value of that property is used.
If not, the next object in the chain is searched for the property, ad nauseum.

In top-level code the scope chain consists of a single object: the global object.
All variables are looked up in the global object, and if it doesn't exist, it is undefined.
In a function the scope chain consists of two objects: the function's call object and the global object.
Variable reference in a function checks call object (local scope) first, then the global object (global scope).
A nested function has three or more objects in its scope chain.

Chapter 5 - Expressions and Operators

An expression is a phrase of JS that an interpreter can evaluate to produce a value (literals and variable names).
Literals: numeric, string, boolean, null, regular expression, object, array, function
The value of a literal expression is the literal value itself.
The value of a variable expression is the value the variable contains or the value to which it refers.

5.2 - Operator Overview

5.2.1 - Number of Operands

Ternary operator ?: combines the value of three expressions into a single expression

5.2.2 - Types of Operands

Although strings cannot be multiplied ("a"*"b" illegal) JS will convert to appropriate type when possible:
"3"*"5" is legal and will evaluate to the number 15 (not the string"15"
When + operator passed a string and a number, the number will be converted to a string and concatenated
lvalue is a historical term meaning "an expression that can legally appear on the lefthand side of an assignment expression"
lvalues: variables, properties of objects, elements of arrays
ECMAScript specification allows built-in functions to return lvalues (but none are defined)

5.2.3 - Operator Precedence

Assignment has very low precedence. Parentheses to override operator precedence.

5.2.4 - Operator Associativity

Unary, assignment, and ternary conditional operators have right-to-left associativity

5.3 - Arithmetic Operators

Object operands are converted to numbers or strings that can be added or concatenated, performed by the valueOf() method and/or the toString() method of the object.
Division: Division by integers yields a float (all numbers are converted to float)
Division by zero yields positive/negative infinity. 0/0 evaluates to NaN

5.2 - Equality Operators

The identity operator (===) checks if two operands are identical (strict definition of sameness).
The equality operator (==) checks whether operands are equal (relaxed definition of sameness).
Numbers, strings, boolean values compared by value.
Objects, arrays, functions compared by reference: equal only if they refer to the same object.
Two separate arrays are never equal or identical, even if they contain equal/identical elements.
Testing two distinct objects or arrays for same properties or elements, respectively, requires checking properties or elements individually.

    Testing for identical values using ===
  • Different types are not identical
  • Identical number values are identical unless NaN (NaN !== NaN). Use NaN() function to test.
  • Strings with identical characters in identical positions are identical.
  • Equivalent boolean values are identical
  • Identical if refer to same object, array, function. Objects/arrays with identical properties/elements not identical.
  • If both values null or undefined they are identical.
    Testing for equality using ==
  • Can be of different types:
    • If null and undefined, they are equal.
    • Convert string to number when comparing to a number
    • Convert bool to number to compare to number
    • Convert objects to primitive types to compare to string (toString() method or valueOf() method

5.4.1.1 - Equality and inequality in Netscape

Never use language="JavaScript1.2" attribute to embed client-side JS code in Netscape, as the equality/nonequality operators behave as identity/non-identity operators.

5.5.1 - Comparison Operators

The String.localeCompare() method takes locale-specific defintions of "alphabetical order" into account.

5.5.3 - The instance Operator

Expects a lefhand operand: object. Expects righthand operand: name of class of objects.
True if lefhand object instance of righthand class. False otherwise.
Classes of objects defined by constructor function used to initialize them.
Righthand operator of instanceof should be name of constructor function.
All objects are instances of Object 

	var d = new Date();	// Create a new object with the Date() constructor
	d instanceof Date;	// Evaluates to true; d was created with Date()
	d instanceof Object;	// Evaluates to true; all objects are instances of Object
	d instanceof Number;	// Evaluates to false; d is not a Number object
	var a = [1, 2, 3];	// Create an array with array literal syntax
	a instanceof Array;	// Evaluates to true; a is an array
	a instanceof Object;	// Evaluates to true; all arrays are objects
	a instaceof RegExp;	// Evaluates to false; arrays are not regular expressions

If lefthand operand not an object or righthand operand object not a constructor of function, returns false.
If righthand operand not an object, runtime error occurs.

5.8 - Bitwise Operators

Bitwise operators require numeric operands with integer values (despite fact all JS numbers float)
Operate on integer operands using 32-bit integer representation.
Some (four of them) perform Boolean algebra on individual bits of operand.
The other (three) are used to shift bits left and right.
Return NaN if used with operands not int, or too large to fit in 32-bit representation in JS 1.0/1.1.
JS 1.2/ECMAScript corece operands to 32-bit int by dropping fractional part of operand (or any bits beyond the 32nd).
Shift operands require righthand operand between 0 and 31.
After converting operand to 32-bit int any bits beyond 5th dropped.

Bitwise AND (&): Bit set in result only if bit is set in both operands, 0x1234 & 0x00FF evaluates to 0x0034
Bitwise OR (|): A bit is set in the result if the corresponding bit is set in one or both of the operands: 9 | 10 evaluates to 11
Bitwise XOR (^): exclusive OR operation, either operand one is true, or operand two is true, but not both.
Bitwise NOT (~): reverses all bits in the operand. Equivalent to changing sign and subtracting 1.
Shift left (<<): moves all bits in 1st operand to left by number of places specified in 2nd operand.
Shift right with sign (>>): moves all bits in 1st operand to right by number of places specified in 2nd operand.
Shift right with zero fill (>>>): Bits shifted in on the left are always zero regardless of 1st operand sign.
