Chapter 2 - Lexical Structure

2.1 - Character Set

JS uses Unicode char set as it can represent virtually every written language in common use.
Each character is represented as 2 bytes as opposed to 8 bits in other languages.

2.2 - Case Sensitivity

JS is a case-sensitive language.

2.3 - Whitespace and Line Breaks

JS ignores spaces, tabs, and newlines between tokens (except those in strings/regular expression literals.

2.4 - Optional Semicolons

Semicolons may be omitted if each statement is placed on a seperate line:

a = 3; a = 4;

is equivalent to:

a = 3 a = 4

However, for the following statement:

a = 3; b = 4;

the first semicolon is required. It is a good programming practice to use semicolons liberally
JS will insert semicolons when it thinks one was omitted if lines of code are inappropriately broken:

return true;

will be interpreted as:

return; true;

2.5 - Comments

C/C++ style comments (// and /* */) are supported in JS.

2.6 - Literals

Literals are data values that appear directly in the program, and include the following:
Integrals, floats, "strings" or 'strings', bool, regular expressions (/javascript/gi), and null.
Expressions that serve as array and object literals are supported in ECMAScript v3:

{ x:1, y:2 }
[1,2,3,4,5]

2.7 - Identifiers

Follows same rules as other languages:
Must start with letter, underscore, or dollar (but avoid these)
ECMAScript v3 can contain letters and digits from complete Unicode char set.
It also allows escape characters (\u03c0)

2.8 - Reserved Keywords

Reserved JavaScript keywords:
break, case, catch, continue, default, delete, do, else, false, finally, for, function, if, in, instanceof, new, null, return, switch, this, throw, true, try, typeof, var, void, while, with


Words reserved for ECMA extensions:
abstract, boolean, bye, char, class, const, debugger, double, enum, export, extends, final, float, goto, implements, import, int, interface, long, native, package, private, protected, public, short, static, super, synchronized, throws, transient, volatile

ECMAScript v4 potential reserved words:
as, is, namespace, use

Other identifiers to avoid (global variables and predefined functions) in ECMAScript v3:
arguments, Array, Boolean, Date, decodeURI, decodeURIComponent, encodeURI, Error, escape, eval, EvalError, Function, Infinity, isFinite, isNaN, Math, NaN, Number, Object, parseFloat, parseInt, RangeError, ReferenceError, RegExp, String, SyntaxError, TypeError, undefined, unescape, URIError

Chapter3 - Data types and Values

Primitive data types supported:
numbers, strings, bool

Trivial data types (single value) supported:
null, undefined

Supports composite data type:
object
An object represents a collection of values (primitive or composite).
Objects can represent unordered collection of named values or ordered collection of numbered values (arrays).

Supports function object:
an object with executed code associated
Functions are invoked to perform an operation.

Classes of objects supported:
Date class: defines objects that represent dates
RegExp class: defines objects that represent regular expressions
Error class: defines objects that represent syntax and runtime errors

3.1 - Numbers

No distinction made between integer and float values, all numbers represented as float.
Numbers represented using 64-bit floating-point format defined by IEEE 754 standard.
Range: +/-5x10^-324 to +/-1.7976931348623157x10^308 (double type range).

3.1.1 - Integer Literals

Values between 9007199254740992 (-2^53) and 9007199254740992 (2^53) inclusive.

Certain integer operations (i.e. bitwise operators) performed on 32-bit ints, -2147483648 (-2^31) to 2147483647 (2^31-1

3.1.2 - Hexadecimal and Octal Literals

Hexadecimal (base-16) values recognized.

0xff // 15 * 16 + 15 = 255 (base 10)
0xCAFE911

ECMAScript standard does not support octal (base-8) format, but some implementations of JS do.

0377 // 3 * 64 + 7 * 8 + 7 = 255 (base 10)

3.1.3 - Floating-Point Literals

floats are real numbers/values: integral.fractional
floats can be represented using exponential notation: [integral][.fractional](E|e)[(+|-)]integerExponent]
Only 18,437,736,874,454,810,627 real numbers can be represented exactly in JS

3.1.4 - Working with Numbers

Basic arithemetic operators: + - * /

Mathematical functions provided by Math object:

sine_of_x = Math.sin(x); hypot = Math.sqrt(x * x + y * y);

The toString() method converts integers to strings using radix (base: 2-36) specified by argument

// Binary Conversion var x = 33; var y = x.toString(2); // y is "100001" // When invoking toString() method on number literal, parentheses can be used to distinguish scope resolution operator from decimal var y = (257).toString(0x10); // y is "101"

3.1.5 - Special Numeric Values

Infinity is a special value representing floating-point values larger than the largest representable finite number
-Infinity (negative infinity) represents floats smaller than the smallest possible finite number

NaN (not-a-number) is an undefined result or error (such as 0 / 0)
NaN does not compare equal to any number, including itself
It is therefore tested with a special function isNaN()
isFinite() tests whether a number is not NaN and not positive or negative infinity

Infinity - Special value to represent infinity
NaN - Special not-a-number value
Number.MAX_VALUE - Largest representable number
Number.MIN_VALUE - Smallest (closest to zero) representable number
Number.NaN - Special not-a-number value
Number.POSITIVE_INFINITY - Special value to represent infinity
Number.NEGATIVE_INFINITY - Special value to represent negative infinity

3.2 - Strings

JS does not use char, use a string instead

3.2.1 - String Literals

Double-quote characters can be contained within single-quote delimited strings
Single-quote characters can be contained within double quote delimited strings
String literals must be written on a single line
Choosing a standard of single or double quotes for JavaScript or HTML is helpful:

<a href="" onclick="alert('Thank you')">Click Me</a>

3.2.2 - Escape Sequences in String Literals

Escape characters are used in JS as they are in other languages
Latin-1 characters can be specified with \xXX where XX are hex digits or \XXX where XXX are octal digits between 1 and 377 (not supported by ECMAScript v3)
Unicode characters can be specified by \uXXXX where XXXX are hex chars
Other escape sequences:
\0 (\u0000)
\b (\u0008)
\t (\u0009)
\n (\u000A)
\v (\u000B)
\f (\u000C)
\r (\u000D)
\" (\u0022)
\' (\u0027)
\\ (\u005C)
A backslash not followed by one of the above characters is ignored.

3.2.3 - Working with Strings

String concatenation supported
JS strings and arrays 0-indexed
Strings use object syntax for accessing properties and methods but are not objects.
String length can be determined with length property, accessed as: s.length (s is a string id)
get last character of string s: last_char = s.charAt (s.length - 1)
extract characters from string s: sub = s.substring(1,4);
find position of first letter a in string s: i = s.indexOf('a');
Some implementations support reading individual characters from strings using array notation: last_char = s[s.length - 1]; however this is not supported by ECMAScript v3 and should be avoided

3.3 - Boolean Values

true or false, test: a == 4
Although bool is distinct type instead of simulated as in C, it is easily converted to other data types and is not as pure as Java boolean data type.

3.4 - Functions

Function: piece of executable code, user defined or predefined 

	function square(x) // x is parameter (argument)
	{
		return x*x;
	}

Functions are values that can be manipulated by JS code: they can be stored in variables, arrays, and objects and they can be passed as arguments to other functions.
When a function is assigned to a property of an object it is referred to as a method of that object.

3.4.1 - Function Literals

Function literals are defined in a way similar to function definitions except they need not have names:
function square(x) { return x*x; }
can be defined as
var square = function(x) { return x*x }
without a name.
Functions defined nameless as above are sometimes called lambda functions in homage to LISP programming language which first allowed unnamed functions to be embedded as literal data values.
The argument list and function body can be passed as strings to the Function() constructor:
var square = new Function("x", "return x*x;");
The above technique is often awkward and less efficient than the other ways to define a function.

3.5 - Objects

An object is a collection of named values (properties or sometimes fields)
Properties of objects are referred to as: [object].[propertyName]
Examples: image.width, image.height
Object properties, like JS variables, can contain any type of data including arrays, functions, and other objects.
e.g. document.myform.button in which the button property of an object is stored in the myform property of an object named document.
When a function value is stored in a property of an object it is called a method, the property name becoming the method name.
The method of an object is invoked using the member access operator (.) to extract function value from object
The function is evoked using the () syntax. Invoking the write() method of the Document object:
document.write("this is a test");
Objects can serve as associative arrays (associate arbitrary data values with arbitrary strings.
A string containing the object property name is enclosed in brackets to access the object properties:
image["width"] image["height"]

3.5.1 - Creating Objects

Constructor functions are used to create objects 

	var o = new Object();
	var now = new Date();
	var pattern = new RegExp("\\sjava\\s", "i");

Once a custom object has been created the properties can be used and set: 
	var point = new Object();
	point.x=2.3;
	point.y = -1.2;

3.5.2 - Object Literals

Also called object initializers, object literals contain a comma-separated list of colon-separated property/value pairs all enclosed in curly braces: 

	var point = { x:2.3, y:-1.2 };

Object literals can also be nested: 
	var rectangle = { upperLeft: {x: 2, y: 2 },
										lowerRight: { x: 4, y: 4 }
									};

The property values used in object literals need not be constant (can be arbitrary expressions): 
	var square = { upperLeft: { x:point.x, y:point.y },
									lowerRight: { x:(point.x + side), y:(point.y + side) }};
